# Project Governance - SOLO CORN SKILLS

## Overview

This document describes the project governance structure, decision-making process, and roles.

## Philosophy

SOLO CORN SKILLS follows a **benevolent dictator** model with community input. The goal is to maintain focus on the project's core mission while encouraging participation.

---

## Roles

| Role | Responsibilities |
|------|-------------------|
| **Maintainer (ceaserzhao)** | Final decision authority, roadmap, releases, merging PRs |
| **Contributors** | Code contributions, documentation, issue reporting, reviews |
| **Community Members** | Feedback, discussion, use cases |

---

## Decision-Making Process

### Major Decisions
- **What counts**: Architecture changes, new core skills, breaking changes
- **Process**: 
  1. Open an issue for discussion
  2. Community feedback (minimum 7 days)
  3. Maintainer decision
  4. Document decision

### Minor Decisions
- **What counts**: Bug fixes, documentation improvements, non-breaking changes
- **Process**: 
  1. Pull request
  2. Code review (1+ approval)
  3. Merge

---

## Version Release Process

### Versioning Scheme
We follow **Semantic Versioning (SemVer)**:
- `MAJOR`: Breaking changes
- `MINOR`: New features, backward compatible
- `PATCH`: Bug fixes

### Release Cadence
- **Minor releases**: Every 1-3 months
- **Patch releases**: As needed
- **Major releases**: Planned annually

### Release Steps
1. Update [CHANGELOG.md](CHANGELOG.md)
2. Update documentation
3. Create a GitHub release with release notes
4. Announce release

---

## Contribution Tiers

### 1. First-time Contributor
- Fix small issues, update documentation

### 2. Regular Contributor
- Consistent contributions over time
- Review other PRs

### 3. Trusted Contributor
- Merge access to documentation
- Vote on minor decisions
