# Preset Mental Model Packages

Curated combinations of mental models optimized for specific decision-making scenarios. Each package addresses a distinct context where multiple models work synergistically to improve judgment quality.

---

## Package 1: Strategic Decision-Making

**Use When**: Evaluating major business decisions, investments, or long-term commitments.

### Included Models

| Model | Primary Function | Why It Fits |
|-------|------------------|-------------|
| First Principles Thinking | Deconstructs problems to fundamental truths | Eliminates inherited assumptions that may no longer apply |
| Second-Order Thinking | Considers consequences of consequences | Prevents nearsighted decision-making |
| Inversion | Approaches problems backward | Reveals failure modes invisible from the forward view |
| Occam's Razor | Cuts through complexity to simplest explanation | Guards against overcomplicating solutions |
| Hanlon's Razor | Attributes outcomes to incompetence rather than malice | Reduces unnecessary conflict and misallocated blame |

### Application Sequence

1. **Frame the problem** using First Principles to identify what you actually know
2. **Map consequences** with Second-Order Thinking (2-3 levels deep)
3. **Stress-test the solution** via Inversion—what could make this fail?
4. **Simplify** using Occam's Razor once viable options emerge
5. **Interpret others' behavior** through Hanlon's Razor to maintain collaboration

### Success Indicators

- Reduced assumption debt (explicitly stated vs. implicit)
- Fewer surprise second-order effects
- Alignment among stakeholders on actual problem definition

## Adapting Packages to Context

These packages are starting points, not rigid prescriptions. Adapt them using these principles:

1. **Add models from adjacent packages** when context overlaps
2. **Remove models** where cognitive load exceeds benefit
3. **Reorder application sequence** based on decision timeline
4. **Combine models** where synergy exceeds individual contribution
5. **Customize** to your domain expertise level and experience

The goal is improved judgment quality, not comprehensive coverage. Use what works for your specific context.
