# Cross-Reference System for Mental Model Combinations

A structured framework for understanding relationships between mental models, enabling informed decisions about which models amplify or conflict when applied together.

---

## Relationship Types

### Synergistic Combinations

Models that enhance each other's effectiveness when combined. The combined effect exceeds the sum of individual applications.

### Complementary Combinations

Models that address different aspects of a problem without redundancy. They cover gaps in each other's analytical approach.

### Sequential Combinations

Models that naturally follow each other in an analytical workflow. Output from one becomes input for the next.

### Conflictual Combinations

Models that can produce contradictory conclusions or interfere with each other's application under certain conditions.

## Sequential Workflow Cross-References

### Decision Quality Enhancement Sequence

```
1. First Principles → 2. Inversion → 3. Second-Order Thinking → 4. Occam's Razor → 5. Bayesian Update
```

| Transition | Rationale |
|------------|-----------|
| First Principles → Inversion | Deconstructed elements reveal hidden inversion opportunities |
| Inversion → Second-Order Thinking | Inversion surfaces failure modes; Second-Order explores their consequences |
| Second-Order → Occam's Razor | Multiple consequences simplify to essential factors |
| Occam's → Bayesian Update | Simplified factors update prior probabilities more reliably |

### Risk Assessment Sequence

```
1. Margin of Safety → 2. Regression to Mean → 3. Asymmetric Betting → 4. Optionality Check → 5. Feedback Loop Monitoring
```

| Transition | Rationale |
|------------|-----------|
| Margin → Regression | Safety margin depends on expected regression direction and magnitude |
| Regression → Asymmetric | Asymmetric positioning accounts for regression timing uncertainty |
| Asymmetric → Optionality | Optionality preserves asymmetric upside while containing downside |
| Optionality → Feedback | Feedback loops reveal when optionality value changes |

### Learning Acceleration Sequence

```
1. Circle of Competence → 2. Learning Curve → 3. First Principles (domain) → 4. Kaizen → 5. Compound Growth Check
```

| Transition | Rationale |
|------------|-----------|
| Circle → Learning Curve | Knowing the circle sets realistic improvement benchmarks |
| Learning Curve → First Principles | Understanding improvement rate focuses principles work on high-leverage areas |
| First Principles → Kaizen | Core mechanisms identified; Kaizen targets improvement at these mechanisms |
| Kaizen → Compound | Small improvements compound when mechanisms are correctly identified |

### System Intervention Sequence

```
1. Feedback Loops Mapping → 2. Reinforcement/Balancing Classification → 3. Leverage Point Identification → 4. Emergence Check → 5. Stock Impact Assessment
```

| Transition | Rationale |
|------------|-----------|
| Feedback → Reinforcement/Balancing | Classification determines intervention strategy |
| Reinforcement/Balancing → Leverage Points | Leverage points differ between loop types |
| Leverage Points → Emergence Check | Emergence may invalidate component-level leverage assumptions |
| Emergence → Stock Assessment | Emergent properties often involve stock accumulation/depletion |

## Category-Level Cross-References

### Decision-Reasoning Models

These models primarily support judgment formation and problem definition.

**Natural Combinations:**
- First Principles + Inversion + Second-Order Thinking
- Bayesian Thinking + Regression to Mean
- Occam's Razor + Hanlon's Razor

**Category Integration:**
Decision-reasoning models excel when applied before execution models. Define the problem correctly (First Principles), map its structure (Feedback Loops), and only then design the solution.

### Execution-Learning Models

These models support implementation, improvement, and capability development.

**Natural Combinations:**
- Kaizen + PDCA Cycle + Feedback Loops
- Compound Growth + Optionality
- Margin of Safety + Learning Curve

**Category Integration:**
Execution-learning models work best in iterative contexts. Apply OODA Loop or PDCA Cycle to create the repetition these models require.

### Psychology-Behavior Models

These models explain human behavior and protect against cognitive errors.

**Natural Combinations:**
- Confirmation Bias Awareness + Falsification Mindset
- Loss Aversion + Commitment Consistency
- Authority Bias Awareness + Incentive Structures

**Category Integration:**
Psychology-behavior models should be applied continuously while using other models. They prevent the application of sound models from being undermined by cognitive distortions.

### Systems-Complexity Models

These models address interconnectedness and emergent behavior.

**Natural Combinations:**
- Feedback Loops + Reinforcement Loops + Balancing Loops
- Tipping Points + Chaos Theory
- Emergence + Stocks and Flows

**Category Integration:**
System models provide the meta-framework for other models. When interventions fail unexpectedly, return to system dynamics to understand why.

### Strategy-Competition Models

These models analyze competitive dynamics and strategic positioning.

**Natural Combinations:**
- Porter's Five Forces + Competitive Moat
- Game Theory + Incentive Structures
- Asymmetric Betting + First-Mover Advantage

**Category Integration:**
Strategy models apply system thinking to competitive contexts. Map the competitive system (Five Forces), identify your position (Moat), then design strategic moves (Game Theory).

## Application Context Cross-References

### When Uncertainty Is High

**Primary Models:** Bayesian Thinking, Asymmetric Betting, Optionality, Margin of Safety

**Support Models:** Chaos Theory, Feedback Loops, Regression to Mean

**Sequence:** Bayesian priors → Asymmetric positioning → Optionality preservation → Feedback monitoring for uncertainty resolution

**Avoid:** Overconfident application of deterministic models (pure Game Theory without uncertainty modeling, linear extrapolation)

### When Time Pressure Is High

**Primary Models:** OODA Loop, First Principles (abbreviated), Two Boxes Thinking

**Support Models:** Inversion (quick failure mode scan), Authority Bias Awareness

**Sequence:** Two Boxes (separate controllable) → First Principles (rapid deconstruction) → OODA acceleration → Inversion (sanity check)

**Avoid:** Extended analysis phases; apply Occam's Razor ruthlessly

### When Stakeholders Are Involved

**Primary Models:** Incentive Structures, Commitment Consistency, Social Proof, Authority Bias Awareness

**Support Models:** Game Theory, Loss Aversion, Second-Order Thinking (stakeholder effects)

**Sequence:** Map incentives → Identify commitment constraints → Apply loss aversion framing → Model stakeholder game theory

**Avoid:** Naive assumptions about aligned interests; Hanlon's Razor should not override legitimate incentive misalignment

### When Past Performance Is Being Evaluated

**Primary Models:** Survivorship Bias Awareness, Regression to Mean, Learning Curve

**Support Models:** Bayesian Thinking, Feedback Loops, Compound Growth

**Sequence:** Survivorship audit → Set regression expectations → Apply learning curve correction → Bayesian update

**Avoid:** Extrapolating past extremes without regression consideration; ignoring sample bias in success stories

### When Building Something New

**Primary Models:** First Principles, Inversion, Emergence, Optionality

**Support Models:** Feedback Loops, Asymmetric Betting, Circle of Competence

**Sequence:** First Principles reconstruction → Inversion for constraints → Emergence check → Optionality preservation

**Avoid:** Premature optimization (violates optionality); ignoring system effects that emergence captures

### When Optimizing Something Existing

**Primary Models:** Kaizen, PDCA Cycle, Feedback Loops, Leverage Points

**Support Models:** Learning Curve, Compound Growth, Balancing Loops

**Sequence:** Map feedback loops → Identify leverage points → Apply Kaizen iterations → Monitor for balancing loop activation

**Avoid:** Disruption for its own sake; revolutionary changes that destroy accumulated stocks

## Using This Cross-Reference System

### When Combining Models

1. **Check the matrix** for known synergies and conflicts
2. **Verify sequential logic** if applying models in workflow
3. **Consider category integration** for comprehensive coverage
4. **Match to application context** for optimal selection
5. **Apply psychology models continuously** to prevent cognitive distortion

### When Experiencing Analytical Conflicts

1. **Identify which models are conflicting** using the matrix
2. **Determine the context** (uncertainty level, time pressure, stakeholder involvement)
3. **Apply context-weighting** to determine which model takes priority
4. **Document the conflict** for future reference and model refinement
5. **Consider sequential resolution** where models apply at different phases

### When Building Custom Packages

1. **Start with a clear decision context** from the Application Context section
2. **Select 4-6 primary models** that cover different analytical angles
3. **Verify cross-references** between selected models
4. **Define a workflow sequence** for application order
5. **Identify support models** for specific sub-problems
6. **Test against historical decisions** for validation

## Relationship Legend

- **Synergistic (S):** Models enhance each other; combine when both are relevant
- **Complementary (C):** Models address different aspects; combine for comprehensive coverage
- **Sequential (Seq):** Models naturally follow each other; apply in indicated order
- **Caution (Ca):** Models can conflict; apply with awareness of potential contradictions
- **Avoid (A):** Models conflict fundamentally; generally do not combine
