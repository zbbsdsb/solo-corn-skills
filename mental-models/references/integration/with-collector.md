# Mental Models for Skill Routing

## Overview

Skill routing involves matching tasks or challenges to the most appropriate capabilities, expertise, or approaches. This integration guide demonstrates how mental models enhance routing decisions by improving problem classification, capability assessment, and match optimization.

## Core Integration Points

### 1. Circle of Competence for Capability Boundaries

Circle of competence provides the foundational framework for routing decisions:

**Boundary Definition:**

- Core competencies: Areas of deep expertise and proven performance
- Adjacent capabilities: Extensions where proficiency exists
- Growth zones: Areas where learning is actively occurring
- Outside boundaries: Areas requiring external support

**Routing Principles:**

- Route critical tasks to core competencies
- Match adjacent tasks to growth zones with oversight
- Externalize tasks outside boundaries or accept elevated risk

### 2. First Principles for Problem Decomposition

Before routing, first principles ensure accurate problem understanding:

**Decomposition Process:**

1. State the problem in its most fundamental form
2. Identify root causes versus symptoms
3. Determine what type of expertise the solution requires
4. Verify whether the decomposed problem matches the perceived problem

**Routing Application:**
Accurate decomposition prevents routing problems to wrong skill domains based on surface-level presentation.

### 3. Second-Order Thinking for Downstream Effects

Routing decisions have downstream consequences:

- Who will this task be routed to after initial handling?
- What dependencies does this routing create?
- How does this routing affect team capability development?
- What happens if the routed party lacks capacity?

### 4. Inversion for Capability Gaps

Inversion reveals routing failures before they occur:

- Instead of asking "who is best for this task?", ask "who might fail this task?"
- Instead of "can we handle this?", ask "what would indicate we cannot?"
- Instead of "is this in our wheelhouse?", ask "what would prove it is outside?"

### 5. Optionality for Routing Strategy

Optionality frameworks optimize routing under uncertainty:

**Option-Preserving Strategies:**

- Route to multiple parties for parallel processing when outcome is uncertain
- Maintain fallback routing paths for critical tasks
- Create routing flexibility that can adapt to emerging information

**Application Contexts:**

- New problem types without established routing patterns
- High-stakes tasks requiring multiple expertise perspectives
- Capacity constraints requiring load distribution

### 6. OODA Loop for Adaptive Routing

OODA loops enable continuous routing optimization:

- **Observe**: Monitor routing outcomes and patterns
- **Orient**: Update understanding of capability-task matches
- **Decide**: Refine routing rules based on evidence
- **Act**: Implement routing changes and measure results

### 7. Feedback Loops for Routing Learning

Effective routing systems require structured feedback:

**Feedback Categories:**

- Outcome feedback: Did the routing produce successful results?
- Process feedback: Was the expertise appropriately matched?
- Efficiency feedback: Was the routing path optimal?
- Learning feedback: What capabilities should be developed?

### 8. Balancing Loops for Load Distribution

Routing must balance competing pressures:

- Load balancing prevents individual burnout
- Capability development requires challenging assignments
- Critical tasks require prioritization over routine work
- Learning opportunities must be distributed equitably

## Capability Assessment Matrix

| Capability Level | Routing Behavior | Oversight Requirement |
|------------------|-----------------|----------------------|
| Expert | Fully autonomous | Standard review |
| Proficient | Autonomous with spot checks | Moderate oversight |
| Competent | Structured guidance | Active oversight |
| Developing | Paired with expert | Close supervision |
| Novice | Learning mode only | Direct supervision |

## Synergistic Combinations

### Circle of Competence + First Principles

Use first principles to classify problems accurately, then match to documented capabilities within established circles.

### OODA Loop + Feedback Loops

Combine rapid decision cycles with structured outcome measurement for continuous routing improvement.

### Optionality + Balancing Loops

Preserve routing flexibility while maintaining equitable distribution across available capabilities.

### Second-Order + Regression to Mean

Avoid overconfidence in early routing success; expect performance to regress toward demonstrated capability levels.

## Key Performance Indicators

- **Match Rate**: Percentage of tasks routed to appropriate capability levels
- **Resolution Time**: Time from routing to outcome completion
- **Rework Rate**: Frequency of rerouting due to initial mismatch
- **Capability Utilization**: Balance across available capacity
- **Learning Transfer**: Evidence of capability development through routing patterns

## Key Takeaways

1. **Circle of competence** establishes clear boundaries for routing decisions
2. **First principles** ensure accurate problem classification before routing
3. **OODA loops** enable adaptive routing based on real-time feedback
4. **Optionality** preserves flexibility when routing uncertainty is high
5. **Balancing loops** prevent burnout while optimizing capability development
6. **Second-order thinking** reveals downstream routing consequences
