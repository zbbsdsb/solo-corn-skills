# Growth Hacking Strategies

## 1. Viral Loop Identification

### 1.1 Viral Loop Framework

```
Viral Loop Components
======================

1. TRIGGER
   [What makes users want to share]

2. ACTION
   [The sharing action itself]

3. REWARD
   [What users get for sharing]

4. EXPRESSION
   [How the share appears]

5. RETENTION
   [How new users are retained]
```

### 1.2 Viral Loop Types

```
Viral Loop Type 1: Word of Mouth
================================
Trigger: Exceptional product experience
Action: Telling friends/family
Reward: Better experience
Example: [Slack, Notion]

Viral Loop Type 2: Incidental Sharing
=====================================
Trigger: Content creation/use
Action: Publishing publicly
Reward: Social validation
Example: [Spotify, Twitter]

Viral Loop Type 3: Incentivized Sharing
=======================================
Trigger: Reward offer
Action: Inviting contacts
Reward: Feature access/credit
Example: [Dropbox, Uber]

Viral Loop Type 4: Embeddable Product
=====================================
Trigger: Product value
Action: Embedding/sharing outputs
Reward: Exposure for product
Example: [YouTube, Medium]

Viral Loop Type 5: Network Effects
==================================
Trigger: More users
Action: Inviting colleagues
Reward: Better product experience
Example: [LinkedIn, Zoom]
```

### 1.3 Viral Coefficient Calculation

```
Viral Coefficient (K-Factor)
=============================

K = Invitations Sent × Conversion Rate

Example:
- Each user sends 2 invitations
- 20% of invitations convert to signups
- K = 2 × 0.20 = 0.40

Target: K > 1.0 (viral)
Current: K = 0.40
Gap: 0.60

Improvement Options:
1. Increase invites per user
   [Current: 2] → [Target: 3]
   
2. Improve conversion rate
   [Current: 20%] → [Target: 35%]

3. Both improvements combined
   [Target K: 1.05]
```

## 2. Referral Program Templates

### 2.1 Referral Program Structure

```
Referral Program Framework
==========================

PROGRAM NAME: [Creative name]

OFFER - REFERER (Giver):
- [Reward for successful referral]
- Example: $20 credit

OFFER - REFERRED (Receiver):
- [Welcome incentive]
- Example: 20% off first order

MECHANICS:
- How many referrals allowed
- Pacing (unlimited vs limited)
- Expiration on rewards

CHANNELS:
- Email invite
- Social sharing
- Unique referral link
- In-app prompt

TRACKING:
- Unique referral codes
- Attribution tracking
- Analytics dashboard

COMMUNICATION:
- Invite message template
- Follow-up message
- Reward notification
```

```

=====================================
What: Co-create content

Partnership Type 2: Promotion Partnership
=========================================
What: Cross-promote to audiences
Who: Non-competing brands
Example: [Product] + [Non-competing SaaS]

==========================================
What: Technical integration
Who: Complementary tools

=========================================
```

## 4. Community Building

### 4.1 Community Strategy

```
Community Building Framework
============================

1. PURPOSE
   - Why does community exist?
   - What value does it provide?
   - Who is it for?

2. PLATFORM
   - Where will community live?
   - [Discord/Slack/Facebook/Forum]

3. PROGRAMS
   - What activities/events exist?
   - How do members engage?

4. PEOPLE
   - Who leads community?
   - Who are supermembers?
   - How to identify/empower them?

5. METRICS
   - How to measure success?
   - What KPIs to track?
```

### 4.2 Community Engagement Tactics

```
Engagement Tactics
==================

DAILY:
- Respond to all comments/messages
- Share user-generated content
- Feature community members
- Post valuable content

WEEKLY:
- Host live Q&A/discussion
- Share weekly roundup
- Spotlight member achievements
- Launch discussion threads

MONTHLY:
- Virtual event/webinar
- Community challenge
- New member welcome
- Product update showcase

QUARTERLY:
- Community survey
- Virtual summit
- Award/showcase event
- Strategy review
```

## 5. Product-Led Growth

### 5.1 PLG Framework

```
Product-Led Growth Strategy
============================

CORE PRINCIPLES:
1. Product is the primary driver of growth
2. User experience determines adoption
3. Data informs product decisions
4. Retention comes from value

PLG PLAYBOOK:

Activation
---------
- Optimize time-to-value
- Create onboarding flows
- Implement product tours
- A/B test onboarding

Expansion
---------
- Identify expansion signals
- Create upgrade paths
- Implement in-app upgrades
- Optimize feature discovery

Retention
--------
- Measure engagement
- Create habit loops
- Implement check-ins
- Celebrate retention milestones
```

### 5.2 PLG Metrics

```
Product-Led Growth Metrics
===========================

Activation Metrics:

- Feature adoption
- Onboarding completion

Engagement Metrics:
- DAU/MAU ratio
- Session frequency
- Feature usage
- Power user identification

Expansion Metrics:
- Upgrade rate
- Expansion revenue
- Seat utilization
- Add-on adoption

Retention Metrics:
- Cohort retention
- Churn rate
- LTV prediction
- Engagement score
```
