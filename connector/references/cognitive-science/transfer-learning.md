# Transfer of Learning

## Origin

**Thorndike & Woodworth** (1901), "The Influence of Improvement in One Mental Function"
Modern research: Perkins & Salomon (1990s)

## Core Concept

Learning in one context can help performance in different contexts.

### Key Points

- **Near transfer**: Similar tasks share skills easily
- **Far transfer**: Different domains require explicit effort
- **Deep structure > Surface similarity**: Transfer depends on underlying principles

## Types of Transfer

### Near Transfer
```
[Math: Algebra] → [Physics: Calculations]
Similar elements: symbolic manipulation, logical reasoning
```

### Far Transfer
```
[Chess: Strategy] → [Business: Competitive positioning]
Similar structure: anticipate opponent moves, think ahead
```

## Research Insights

### What Helps Transfer

1. **Abstract principles**: Understand WHY, not just HOW
2. **Multiple contexts**: Practice in varied situations
3. **Explicit connections**: Make similarities visible
4. **Mental models**: Deep structures transfer better

### What Hinders Transfer

1. **Over-specific learning**: "How to" without "why"
2. **Fixed mindset**: Believing skills are domain-specific
3. **Surface focus**: Attending to surface features only
4. **Passive learning**: No active reflection on connections

## Application in Connector

### Connection Discovery Module

The core purpose: Make cross-domain transfer visible and actionable.

**Process**:
```
New Domain (e.g., Blockchain)
    ↓
Identify Deep Structure
(Cryptography + Distributed Systems + Game Theory)
    ↓
Find Known Domains with Same Structure
(Cloud Computing + Voting Systems + Economics)
    ↓
Generate Cross-Domain Analogies
"Blockchain consensus is like..."
```

### Mental Models as Transfer Bridges

Mental models are the **deep structures** that enable far transfer.

| Mental Model | Deep Structure | Transfer Domains |
|--------------|----------------|------------------|
| Second-order thinking | Consequence chains | Strategy, investing, relationships |
| Leverage points | Intervention selection | Policy, systems, product |
| Inversion | Problem reframing | Problem-solving, creativity |
| First principles | Element decomposition | Science, business, design |

## Connector Implementation

### QuickScan: Transfer Scan
```
"After learning [domain], we'll explicitly ask:
- What does this remind you of?
- What's the deep structure?
- Which mental models apply?"
```

### Learning Path: Transfer Exercises
```
Exercise 1: Find 3 connections to domains you know
Exercise 2: Generate cross-domain analogies
Exercise 3: Apply a mental model from another field
Exercise 4: Teach the deep principle to someone else
```

### Expert Prep: Transfer Questions
```
"When talking to an expert, ask:
- What's the deep principle here?
- How is this similar to [known field]?
- What advice would you give someone coming from [background]?"
```

## Key Takeaway

> "The more you see deep structures, the more you can transfer."

## Practical Transfer Checklist

- [ ] Can you explain the DEEP PRINCIPLE (not just surface features)?
- [ ] Have you practiced in MULTIPLE CONTEXTS?
- [ ] Have you explicitly identified CONNECTIONS to known domains?
- [ ] Can you generate your own ANALOGIES?
- [ ] Can you TEACH it to someone from a different background?
