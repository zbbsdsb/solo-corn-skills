# Generation Effect

## Origin

**Slamecka & Graf** (1978), "The Generation Effect," *Journal of Experimental Psychology*

## Core Concept

Information generated by the learner is remembered better than information read.

### Key Points

- **Active > Passive**: Generating answers creates stronger memory than passive review
- **Self-generated connections**: We remember connections we make ourselves
- **Testing effect**: Retrieval practice > re-reading

## Why It Works

1. **Deeper processing**: Generation requires semantic analysis
2. **Meaning making**: We construct understanding actively
3. **Retrieval practice**: Trying to recall strengthens memory traces

## Research Findings

| Method | Retention Rate |
|--------|---------------|
| Re-reading | ~30% |
| Taking notes | ~50% |
| Self-testing | ~70% |
| Generating answers | ~80% |

## Application in Connector

### Strategies

1. **Fill-in-the-Blank**
   ```
   "Quantum bits can exist in multiple states simultaneously. 
   This property is called _______."
   ```

2. **Self-Explanation Prompts**
   ```
   "In your own words, explain why quantum computing 
   might be faster for certain problems."
   ```

3. **Active Recall**
   ```
   "Without looking, what are the 3 main types of 
   machine learning?"
   ```

4. **Connection Making**
   ```
   "How is the concept of 'chunking' in quantum computing 
   similar to something you already know?"
   ```

## Connector Implementation

### QuickScan Prompts
- "Summarize this concept in one sentence"
- "What questions does this raise for you?"
- "Connect this to something you already know"

### Learning Path Exercises
- Generate your own examples
- Teach back what you learned
- Create analogies yourself

## Key Takeaway

> "Don't just read. Generate. Recall. Connect."
