# Situated Learning

## Origin

**Lave & Wenger** (1991), "Situated Learning: Legitimate Peripheral Participation"

## Core Concept

Knowledge cannot be learned independently from its use context.

### Key Points

- **Learning is contextual**: Skills are tied to situations
- **Legitimate peripheral participation**: Learn by doing edge tasks in communities
- **Apprenticeship model**: Learning through guided practice

## Traditional vs. Situated Learning

| Traditional | Situated |
|------------|----------|
| Classroom | Real-world projects |
| Abstract concepts | Embedded in practice |
| Individual study | Community participation |
| Decontextualized | Context-rich |

## Lave & Wenger's Insight

> "Learning is not merely situated in practice—as if it were some independently isolable aspect of culture that could be captured in a textbook—but that it is an integral part of generative social practice in the lived-in world."

## Application in Connector

### 1. Project-Based Learning

Instead of learning "about" a domain → Learn by doing a project IN the domain.

**Before**
```
Read about growth hacking
↓
Learn growth frameworks
↓
Study case studies
```

**After**
```
"I want to grow my SaaS user base"
↓
Learn growth hacking BY launching experiments
↓
Get real feedback
↓
Iterate
```

### 2. Expert Community Integration

- Connect learners to communities of practice
- Facilitate mentorship relationships
- Enable legitimate peripheral participation

### 3. Scenario-Based Learning

**Example: Learning "Blockchain"**
```
Traditional: "Blockchain is a distributed ledger..."
↓

Situated: "Imagine you're evaluating whether to add 
crypto payments to your e-commerce store. What do 
you need to understand?"
```

### 4. Case Study Method

Real situations > hypothetical examples

## Connector Implementation

### QuickScan: Contextual Entry Point
```
"Before we dive in, what's your project? Learning 
blockchain to evaluate a startup investment? Or to 
build a crypto product?"
```

### Learning Path: Project-Based Structure
```
Week 1: Launch your first experiment
Week 2: Analyze results
Week 3: Iterate based on feedback
Week 4: Present to community
```

### Expert Prep: Conversation Context
```
"You're meeting with a blockchain expert. What's 
your specific project context? We'll prepare 
questions relevant to YOUR situation."
```

## Key Takeaway

> "Don't learn about. Learn by doing."

## Practical Framework

### For Each Learning Goal

1. **What's your project?**
   - Define a real-world task
   - Set authentic constraints

2. **Who's in the community?**
   - Find practitioners
   - Identify mentors

3. **What's your edge task?**
   - Start at periphery
   - Progress gradually

4. **How do you reflect?**
   - Document learning
   - Connect to practice
