# Cognitive Load Theory

## Origin

**John Sweller** (1988), "Cognitive Load During Problem Solving," *Cognitive Science*

## Core Concept

Working memory has limited capacity. Learning new material overloads cognitive resources.

### Key Points

- **Working memory capacity**: ~7 ± 2 information chunks
- **Experts compress**: Can chunk more information due to schema
- **Novices cannot**: Must process each element separately

## Three Types of Cognitive Load

### 1. Intrinsic Load
- Inherent complexity of the material
- Cannot be eliminated, only restructured

### 2. Extraneous Load
- Poor instructional design
- Can and should be eliminated

### 3. Germane Load
- Mental effort spent on learning
- Should be maximized

## Application in Connector

### Strategies

1. **Chunking**
   - Break information into meaningful groups
   - Provide conceptual frameworks first

2. **Reduce Extraneous Load**
   - Clear, simple explanations
   - Avoid unnecessary complexity
   - Use consistent formats

3. **Manage Intrinsic Load**
   - Layer information (surface → middle → deep)
   - Build on existing knowledge
   - Provide scaffolding

## Practical Examples

### Before (High Load)
```
Quantum computing involves superposition states where qubits can exist 
in multiple states simultaneously through quantum entanglement which 
enables exponential computational power through...
```

### After (Managed Load)
```
Quantum computing = Faster computers for specific problems

Think: Special calculator for special problems (not replacement for your laptop)
```

## Connector Implementation

- **QuickScan**: Provides high-level framework first, details later
- **Concept Trees**: Visual chunking of domain knowledge
- **Progressive Disclosure**: Surface → Middle → Deep layers
- **Analogies**: Connect new to familiar, reducing intrinsic load

## Key Takeaway

> "Don't overwhelm. Scaffold. Chunk. Connect to what they know."
