# Concept Graph Building Method

## Methodology Description
Based on cognitive network theory, build a knowledge system by visualizing concept relationships.

## What is a Concept Graph

Concept Graph = Nodes (concepts) + Edges (relationships)

## Building Steps

### Step 1: Concept Collection (30 minutes)
- Quickly read materials, mark important concepts
- Don't filter, collect first
- Record frequently occurring terms

**Tips**:
- Mark different types of concepts with different colors
- Annotate concepts you don't understand

---

### Step 2: Relationship Identification (30 minutes)
- What is the relationship between concepts?
  - Inclusion relationship (A includes B)
  - Causal relationship (A causes B)
  - Sequential relationship (A comes before B)
  - Similarity relationship (A is similar to B)
  - Comparison relationship (A vs B)

**Tips**:
- Look for connectives like "because therefore", "first then"
- Ask "What is the premise of this concept?"

---

### Step 3: Hierarchical Organization (30 minutes)
- Find the most core concept (root node)
- Establish hierarchical structure
- Identify important subcategories

**Tips**:
- Ask "If I could only remember one concept, what would it be?"
- Organize using a tree structure

---

### Step 4: Visual Drawing (30 minutes)
- Start from core concept
- Expand outward
- Distinguish with different colors and shapes

**Tool Recommendations**:
- Pen and paper: Fastest, most flexible
- Digital tools: Miro, Notion, Draw.io
- Mind mapping software: XMind, MindNode

---

### Step 5: Iterative Optimization (30 minutes)
- Add missing concepts
- Adjust connection relationships
- Simplify redundant parts

**Tips**:
- Look at it again after 24 hours
- Ask others for feedback
- Continuously update during learning

---

## Graph Types

### 1. Hierarchical Graph
```
Core Concept
├── Subconcept 1
│   └── Grandchild Concept
└── Subconcept 2
```
Suitable for: Subject knowledge, organizational structure

### 2. Flow Graph
```
Concept A → Concept B → Concept C
```
Suitable for: Workflows, causal chains

### 3. Network Graph
```
Concept A ── Concept B
  │         │
Concept C ── Concept D
```
Suitable for: Complex systems, interrelationships

### 4. Comparison Graph
```
  Concept A    vs    Concept B
  Feature 1        Feature 1
  Feature 2        Feature 2
```
Suitable for: Similar concept comparisons

---

## Scientific Principles

### Cognitive Network Theory
Knowledge is stored in the brain in network form; activating one concept spreads to related concepts.

### Dual Coding Theory
Encoding information with both text and visuals creates stronger memories.

### Generation Effect
Actively building graphs works better than passive reading.

---

## Application Scenarios

- New book learning: Draw a graph after each chapter
- Course learning: Supplement graph after each class
- Project planning: Clarify relationships between parts
- Knowledge organization: Regularly update your knowledge network
