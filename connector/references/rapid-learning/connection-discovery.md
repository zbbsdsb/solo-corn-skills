# Connection Discovery Module

## Purpose

Find connections between the new domain and what you already know.

## Theory Base

- **Transfer Learning**: Deep structure > surface features
- **Associative Learning**: New connections strengthen memory
- **Mental Models**: Bridge concepts across domains

## Core Principle

> "Your cross-domain knowledge is your superpower."

## Workflow

```
1. Identify Deep Structures → What are the FUNDAMENTAL patterns?
2. Find Known Analogs → What does this remind you of?
3. Apply Mental Models → Which frameworks transfer?
4. Generate Insights → What does this suggest?
```

## Step 1: Identify Deep Structures

### Prompt

```
For each core concept in [domain], ask:
"What is this REALLY about at a fundamental level?"

Not "What is it?" but "What PATTERN is this an example of?"
```

### Deep Structure Examples

| Surface Domain | Deep Structure |
|----------------|----------------|
| Blockchain | Distributed consensus without trust |
| Machine Learning | Pattern recognition from examples |
| Evolution | Random variation + selection |
| Markets | Voluntary exchange + price signals |
| Evolution | Random variation + selection |

### Template

```markdown
## Deep Structure Analysis

| [Domain] Concept | Surface Description | Deep Structure |
|-------------------|--------------------| ---------------|
| [Concept A] | [What it does] | [What fundamental pattern?] |
```

## Step 2: Find Known Analogs

### Cross-Domain Analogy Framework

```markdown
## [New Domain] ↔ [Known Domain]

### Similarities
1. [Both have X]
2. [Both work because Y]
3. [Both face challenge Z]

### Differences
1. [Domain A has X, Domain B has Y]
2. [Key mechanism differs]

### What Transfers
- [Mental model A] applies because...
- [Strategy B] might work because...
```

### Example

```markdown
## Blockchain ↔ Wikipedia

**Similarities**:
1. Both are shared, visible to all
2. Both improve with contributions
3. Both have versions/change history

**Differences**:
1. Wikipedia can be edited by anyone, blockchain requires consensus
2. Wikipedia can be corrected, blockchain is immutable
3. Wikipedia is voluntary, blockchain is automated

**What Transfers**:
- Understanding that "many eyes make all bugs shallow" applies
- The importance of clear contribution rules
```

## Step 3: Apply Mental Models

### Prompt

```
Which of our 45+ mental models might apply here?

Think:
- Decision frameworks
- Strategy models
- Systems thinking
- Psychology biases
```

### Example Mapping

| New Domain Concept | Mental Model | Why It Fits |
|-------------------|--------------|-------------|
| Blockchain consensus | Second-order thinking | Need to think about incentives 2-3 steps ahead |
| Machine learning bias | Inversion | Start from "what could go wrong?" |
| Tech adoption | Crossing the chasm | Similar to market segmentation |
| Network effects | Flywheel | Positive feedback loops |

## Step 4: Generate Insights

### Connection Exercises

#### Exercise 1: "This reminds me of..."

```markdown
## Cross-Domain Memory

When I learned about [new concept], 
it reminded me of [known concept] because...

This suggests [insight] about [application area].
```

#### Exercise 2: "A pattern I've seen before..."

```markdown
## Pattern Recognition

I've seen the [pattern] before in:
1. [Domain A]: [Example]
2. [Domain B]: [Example]
3. [Domain C]: [Example]

The common principle is: [Deep structure]

This means for [new domain], I should: [Action]
```

#### Exercise 3: "If I were the expert..."

```markdown
## Expert Projection

If I were an expert in [new domain], 
I'd apply what I know from:

1. [Strategy] from [domain] → Because [connection]
2. [Tactic] from [domain] → Because [connection]

This might be novel because [reason].
```

## Key Takeaway

## Practice Prompts

### Daily Connection Practice

```
At the end of each learning session:
"What does this remind me of?"

Force yourself to find at least 3 connections 
before moving on.
```

### Weekly Integration

```
Review this week's learning:
1. What themes emerged across domains?
2. What mental models connected most?
3. What's one insight you couldn't have had alone?
```

### Monthly Synthesis

```
Look at your knowledge network:
1. Which nodes are most connected?
2. What's the bridge between two clusters?
3. What's missing—a connector that would link everything?
```
