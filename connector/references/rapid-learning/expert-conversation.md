# Expert Conversation Preparation Module

## Purpose

Prepare to have effective, intelligent conversations with domain experts.

## Theory Base

- **Curse of Knowledge**: Bridge the expertise gap
- **Generation Effect**: Prepare actively vs. passive reading
- **Situated Learning**: Learn in context of conversation

## Core Principle

> "5% of the knowledge lets you ask 95% of the good questions."

## The 5% Rule

You don't need to be an expert. You need:
- ✓ The right vocabulary
- ✓ The right questions
- ✓ The right context
- ✓ The right humility

## Workflow

```
1. Must-Know Concepts → What you MUST understand
2. Vocabulary Builder → Key terms in plain English
3. Question Strategy → Questions that impress experts
4. Trap Avoidance → Questions that expose ignorance
5. Credibility Signals → How to sound like you get it
```

## Step 1: Must-Know Concepts

### Framework

```markdown
## Must-Know for Expert Conversation

### 1. The Definition (Be able to say)
"In my understanding, [domain] is fundamentally about..."

### 2. The Key Tension (Know the debates)
"The field seems divided on X vs. Y because..."

### 3. The Common Pitfall (Know what's hard)
"Most beginners struggle with [concept] because..."

### 4. The Exciting Part (Know what's interesting)
"What's really fascinating is [recent development]..."
```

### Example: Blockchain

```markdown
## Blockchain Expert Prep

### Definition
"In my understanding, blockchain is fundamentally about 
enabling trust without intermediaries through distributed 
consensus."

### Key Tension
"The field is divided between those who think blockchain 
is for everything vs. those who think it's for specific 
use cases where trust is truly the bottleneck."

### Common Pitfall
"Most beginners think blockchain solves the trust problem, 
but actually it just shifts the trust—you now trust the 
consensus mechanism instead of a bank."

### Exciting Part
"What's really fascinating is how zero-knowledge proofs are 
enabling privacy on public blockchains—this could unlock 
enterprise adoption."
```

## Step 3: Question Strategy

### High-Value Questions

#### Understanding Questions
```markdown
1. "What's the most counterintuitive thing about [field]?"
2. "What do most people get wrong about [topic]?"
3. "If you were starting fresh today, what would you do differently?"
```

#### Opinion Questions
```markdown
1. "What's your controversial opinion about [topic]?"
2. "If you could change one thing about [field], what would it be?"
3. "What's the biggest misconception in your industry?"
```

#### Future Questions
```markdown
1. "Where do you see the field going in 5 years?"
2. "What's an unsolved problem you're excited about?"
3. "What bets are you making that others aren't?"
```

#### Personal Questions
```markdown
1. "What resources would you recommend for someone at my level?"
2. "Who else should I talk to?"
3. "What's the fastest way to learn more?"
```

### Questions That Impress Experts

```markdown
# Questions that make experts light up

1. "What's the most important thing you're NOT working on?"
   → Shows long-term thinking

2. "What's a failure that taught you the most?"
   → Shows you value learning from mistakes

3. "If this [technology/approach] fails, why will it be?"
   → Shows you've thought critically

4. "What does the field need that it doesn't have yet?"
   → Shows you're solving real problems
```

## Step 4: Trap Avoidance

### Questions That Expose Ignorance

```markdown
# AVOID These

✗ "So, how does blockchain work exactly?"
   → Too broad, shows you didn't do homework

✗ "Isn't this just like [completely different thing]?"
   → Shows superficial understanding

✗ "Why hasn't anyone done [obvious thing] yet?"
   → Often has good reasons that reveal ignorance

✗ "Could you explain [basic concept] to me?"
   → Should have learned this beforehand
```

### Better Alternatives

```markdown
# INSTEAD, Ask

✓ "I've read about [specific aspect]. How does that connect to your work?"
   → Shows you did homework

✓ "Is the comparison to [similar thing] valid? Where does it break down?"
   → Shows you've thought about it

✓ "I've wondered about [informed question]. What's your take?"
   → Shows you understand there's nuance
```

## Step 5: Credibility Signals

### What Experts Look For

```markdown
# Credibility Signals

1. **Shows preparation**
   "I read your piece on X. It made me think about Y..."

2. **Asks informed questions**
   "You mentioned Z in your talk. Does that apply to A as well?"

3. **Connects to context**
   "We're facing [specific problem]. How would you approach it?"

4. **Admits limits appropriately**
   "I'm still learning about [aspect]. But I'm wondering about..."
```

### Credibility Killers

```markdown
# AVOID

✗ Pretending to know more than you do
✗ Using jargon incorrectly
✗ Dismissive of expert opinions
✗ Asking questions you could Google
✗ Monopolizing the conversation
```
