# Rapid Mastery Path Module

## Purpose

Generate personalized "just enough" learning paths.

## Theory Base

- **Desirable Difficulties**: Optimal challenge level
- **Spaced Repetition**: Long-term retention
- **Metacognition**: Self-regulated learning

## Core Principle

> "Not 'become an expert.' Become someone who can have an intelligent conversation."

## The 3-Layer Model

### Surface Layer (2 hours)
**Goal**: Understand enough to have a basic conversation

```
✓ Know what it is (one-line definition)
✓ Know the main concepts (3-5)
✓ Know the key terminology
✓ Know why it matters
✓ Can spot related opportunities/problems
```

### Middle Layer (1-3 days)
**Goal**: Apply it to your project

```
✓ Can read advanced content
✓ Can ask intelligent questions
✓ Can evaluate claims
✓ Can make basic decisions
✓ Can work with practitioners
```

### Deep Layer (1 week+)
**Goal**: Become a practitioner

```
✓ Can teach it to others
✓ Can handle edge cases
✓ Can innovate within the field
✓ Can contribute back
✓ Can recognize experts vs. posers
```

```
- [ ] Can ask good questions

### Deep Level
- [ ] Can teach to others

- [ ] Can innovate

```

## Prioritization Framework

### 20/80 Rule for Each Layer

**Surface (2 hours)**:
```
20% of concepts give 80% of understanding
→ Focus on: Core concepts + Key vocabulary
```

**Middle (1-3 days)**:
```
20% of applications give 80% of value
→ Focus on: Most relevant applications + Common patterns
```

**Deep (1 week+)**:
```
20% of practitioners know 80% of what matters
→ Focus on: Expert mental models + Real-world experience
```

## Example Path: Blockchain for Product Manager

```markdown
# Blockchain Mastery Path

**Target**: Surface → Middle (1 week)
**Project**: Evaluate blockchain for product features

## Day 1: Surface Layer (2 hours)

### Blockchain Basics
- [ ] Video: "Blockchain explained in 5 minutes" (10 min)
- [ ] Read: "Blockchain in 100 words" (5 min)
- [ ] Watch: How blockchain works (15 min)

### Key Vocabulary
- [ ] Decentralized
- [ ] Consensus
- [ ] Immutable
- [ ] Smart contract

**Checkpoint**: Can you explain blockchain to a friend?

## Day 2-3: Product Applications

### Use Cases
- [ ] Case study: Supply chain (Amazon, Walmart)
- [ ] Case study: Digital identity
- [ ] Case study: Tokenization

### Product Thinking
- [ ] When does blockchain help vs. hurt?
- [ ] UX implications
- [ ] Regulatory considerations

**Checkpoint**: Can you identify 3 opportunities for blockchain in your product?

## Day 4-5: Deep Dive

### Technical Understanding
- [ ] Wallets and accounts
- [ ] Gas/transaction fees
- [ ] Oracles and off-chain data

### Expert Perspectives
- [ ] Read: "Blockchain Truths" by Chris Dixon
- [ ] Listen: 2-3 podcasts with blockchain founders

**Checkpoint**: Can you evaluate a blockchain startup's claims?

## Day 6-7: Integration

### Your Project Plan
- [ ] Define problem blockchain solves
- [ ] Evaluate build vs. buy vs. partner
- [ ] Create decision framework

**Final Checkpoint**: Present your blockchain strategy to a skeptic.
```

## Key Takeaway

> "Learn the 20% that gives 80% of the value. Then decide if you need more."
