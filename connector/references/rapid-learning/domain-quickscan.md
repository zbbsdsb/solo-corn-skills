# Domain QuickScan Module

## Purpose

Get a complete cognitive map of any domain in 30 minutes.

## Theory Base

- **Cognitive Load Theory**: Layered information delivery
- **Generation Effect**: Active engagement during overview
- **Situated Learning**: Context-first approach

## Workflow

```
1. Context Discovery → Understand WHY
2. Domain Landscape → High-level structure
3. Core Concepts → Essential knowledge
4. Key Players → People and resources
5. Quick Assessment → What you learned
```

## Step 1: Context Discovery

### Questions

```
Before we dive in, tell me:
1. What's your project? (Why do you need to learn this?)
2. What's your background? (What do you already know?)
3. How much time do you have? (30 min / 2 hours / 1 day)
4. What's your goal? (Understand enough to talk / actually apply / become expert)
```

### Output

```markdown
## Learning Context

**Project**: [What you're working on]
**Background**: [Your prior knowledge]
**Time Available**: [Your time constraint]
**Goal**: [Understanding level needed]
```

## Step 2: Domain Landscape

### Template

```markdown
## [Domain] at a Glance

### One-Line Definition
[Explain as if to a 10-year-old]

### What It Is
[2-3 sentences]

### What It's NOT
[Common misconceptions]

### Why It Matters
[Business/personal relevance]
```

### Example

```markdown
## Blockchain at a Glance

**One-Line**: A shared notebook that everyone can read but no one can cheat.

**What It Is**: A way to keep records that many people can verify, without needing a bank or government to guarantee trust.

**What It's NOT**: 
- Not just cryptocurrency
- Not a replacement for all databases
- Not automatically secure/t fast/cheap

**Why It Matters**: Enables trustless transactions, reduces intermediaries, enables new business models.
```

## Step 3: Core Concepts

### Concept Tree Template

```markdown
## Core Concept Tree

### Layer 1: Fundamentals (Learn first)
- [Concept A]
- [Concept B]
- [Concept C]

### Layer 2: Key Mechanisms (How it works)
- [Mechanism 1]
- [Mechanism 2]
- [Mechanism 3]

### Layer 3: Applications (What it's used for)
- [Use Case 1]
- [Use Case 2]
- [Use Case 3]
```

```markdown
## Essential Vocabulary

| Term | Plain English | Why It Matters |
|------|---------------|----------------|
| [jargon] | [simple definition] | [when you'll need it] |
```


### Resources
```markdown

**Best Entry Points**:
- [Resource 1] - [Why it's good]
- [Resource 2] - [Why it's good]

**Common Pitfalls**:
- [Pitfall 1] - [How to avoid]
```

## Step 5: Quick Assessment

### Self-Test

```markdown
## Quick Self-Test

Can you:
1. ✓/✗ Define [domain] in one sentence?
2. ✓/✗ Name the 3 core concepts?
3. ✓/✗ Explain why it matters?
4. ✓/✗ Identify 1 thing you didn't know before?
```

### Reflection Questions

```markdown
## Reflection

1. What surprised you most?
2. What questions remain?
3. What will you learn next?
```

## Desirable Difficulty Insert

At step 3, add prediction exercise:

```
"Before I show you the concept tree, try to 
predict: What do you think are the 3 most 
important concepts in [domain]?

Write your prediction, then we'll compare."
```

## Metacognitive Check

At end:

```
"Rate your understanding now (1-5): ___

What do you know now that you didn't know 
before?

What do you still not understand?"
```
