# Outcome Tracking Template

## Overview

This document provides a structured template for tracking the actual outcomes of decisions against their expected outcomes. It enables continuous learning and improvement of decision-making processes.

---

## Example Entry

```markdown
# Outcome Tracker: PostgreSQL Database Migration

| Field | Value |
|-------|-------|
| **Decision ID** | DEC-2024-002 |
| **Implementation Date** | 2024-02-15 |
| **Review Date** | 2024-05-15 |
| **Reviewer** | Backend Lead |
| **Time Horizon** | 3 months |

## Success Criteria Comparison

### Criterion: Query Performance

| Aspect | Details |
|--------|---------|
| **Expected** | 50% improvement in complex query response time |
| **Actual** | 62% improvement |
| **Variance** | +12% |
| **Status** | Exceeded |

**Analysis**: Complex joins are performing better than expected due to better query planning in PostgreSQL 15.

## Overall Assessment

| Dimension | Score |
|-----------|-------|
| **Outcome Quality** | 5/5 |
| **Process Quality** | 4/5 |
| **Learning Captured** | 5/5 |
| **Overall Score** | 4.67/5 |

**Recommendation**: Continue with PostgreSQL. Document lessons learned for future migrations.
```
