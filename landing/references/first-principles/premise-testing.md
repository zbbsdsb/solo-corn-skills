# First Principles: Premise Testing Framework

## What If This Assumption Is Wrong?

**Core Principle**: Every assumption is a potential point of failure. The strength of your solution is only as good as the assumptions it's built upon.

---

## The Assumption Inversion Exercise

### For Every Assumption, Ask

```
Original Assumption: "_____________________________"

1. What if the OPPOSITE is true?
   → "_____________________________________"

2. What if this assumption is COMPLETELY FALSE?
   → "_____________________________________"

3. What if this assumption is only 50% correct?
   → "_____________________________________"

4. What if this assumption was never valid to begin with?
   → "_____________________________________"
```

## Reverse Thinking Protocol

### The Five Reversal Questions

**Question 1: Inverse Reality**
> What if the opposite of conventional wisdom were true?

| Conventional Wisdom | Inverse Reality | Evidence Needed |
|--------------------|----------------|----------------|
| "More features = better product" | "Simpler = more valuable" | Compare adoption rates |
| "Users know what they want" | "Users don't know what they need" | Observe unarticulated needs |
| "Competition is bad" | "Competition validates market" | Market size analysis |
| "First mover advantage" | "Fast follower wins" | Time-to-market vs quality |

**Question 2: Extreme Reduction**
> What if we could only do ONE thing?

**Exercise**: Strip your solution down to its absolute minimum. What remains?

```
The ONE essential thing: _____________________________

Is this truly essential? How do I know?
_____________________________________________

What am I hiding behind the "essential" label?
_____________________________________________
```

**Question 3: Reversed Causality**
> What if the effect I thought was the cause is actually the result?

| I Thought | What If Instead | Alternative Explanation |
|----------|----------------|------------------------|
| A causes B | B causes A | Correlation vs Causation |
| X is the problem | X is a symptom | What creates X? |
| This constraint is real | This constraint is chosen | Who benefits from it? |

**Question 4: Absurdity Test**
> Push the assumption to its logical extreme

**Example**: If "users won't pay for this," what if it was 100x more valuable? Would they pay then?

```
For my assumption: "____________________"

Push it to 10x extreme: ____________________

Push it to 100x extreme: ____________________

Does the assumption still hold at extreme levels?
_____________________________________________
```

**Question 5: Tabula Rasa**
> Imagine the assumption never existed. What would you have to believe to be true instead?

**Rebuild from Scratch**:

```
Without assuming "[original assumption]",

I would have to assume instead:
1. _________________________________
2. _________________________________
3. _________________________________

Which of these alternative assumptions is more robust?
_____________________________________________
```

## Premise Deconstruction Protocol

### The Five Levels of "Why"

**Purpose**: Dig beneath surface-level assumptions to find foundational truths

```
Level 1: I assume that _____________________________
         Why? _____________________________________
         
Level 2: I assume that _____________________________
         Why? _____________________________________
         
Level 3: I assume that _____________________________
         Why? _____________________________________
         
Level 4: I assume that _____________________________
         Why? _____________________________________
         
Level 5: I assume that _____________________________
         Why? [FUNDAMENTAL TRUTH]
```

### Premise Elimination Hierarchy

**Step 1: Identify all premises**
List every belief your solution rests upon

**Step 2: Categorize premises**
- **Essential**: Solution fails without this
- **Important**: Solution weakened but survives
- **Nice to have**: Solution largely unaffected

**Step 3: Test elimination**
For each essential premise, ask:
- Can this premise be ELIMINATED?
- Can the solution be REDESIGNED to not depend on this?
- What would the solution look like WITHOUT this premise?

**Step 4: Build premise-free alternatives**
```
Premise: "Users need feature X"

Premise-free alternative:
- What if users don't need X?
- What if X is actually harmful?
- What if X could be replaced by Y?

Resulting design: _____________________________
```

## Final Protocol

### The Three-Day Rule

**After making any major assumption, wait three days, then revisit**:

1. Do I still believe this assumption? Why?
2. Has anything challenged this assumption in the interim?
3. What would make me change my mind?

### Assumption Journal

Maintain a running log:

```
Date: [Assumption]: [Status after reflection]: [Changed? Y/N]: [If yes, how]
```

---

*Remember: The goal is not to doubt everything—it's to separate strong knowledge from weak opinions, and build accordingly.*
