# Problem Statement Extraction Template

## Overview

The Problem Statement Extraction Template is designed to transform ambiguous user descriptions into clear, actionable problem statements. This template emphasizes the critical importance of the clarification process in reducing misunderstandings and ensuring project alignment.

**Why Clarification Matters:**
- Prevents costly rework by identifying misalignments early
- Ensures all stakeholders share a common understanding of the problem
- Creates a foundation for measurable success criteria
- Reduces project scope creep through precise problem definition


### 3.2 Problem Framing Options

**Option A - Problem-Centered Framing:**
```
[Current problematic state] prevents [stakeholder] from achieving 
[desired outcome], resulting in [measurable negative impact].
```

**Option B - Opportunity-Centered Framing:**
```
[Stakeholder] could achieve [desired outcome] by [X%], if we 
address [the problem], enabling [positive impact].
```

**Option C - Outcome-Centered Framing:**
```
To achieve [specific outcome], we need to solve [problem], which 
currently affects [scope] and causes [impact].
```

## Usage Guidelines

**When to Use This Template:**
- During initial requirements gathering
- When encountering ambiguous or complex user requests
- Before proceeding with solution design
- When misalignment is suspected

**Key Questions to Ask During Extraction:**
1. What specific pain point are you experiencing?
2. How does this problem currently impact your work?
3. What would success look like if this problem were solved?
4. Who else is affected by this problem?
5. What have you already tried to solve this?

**Red Flags to Watch For:**
- Vague problem descriptions ("improve efficiency")
- Solution-first thinking ("we need to build X")
- Scope that keeps expanding during discussion
- Multiple conflicting problem statements from different stakeholders
- Problem statements that lack measurable impact
